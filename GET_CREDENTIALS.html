Step-by-Step Guide

Step 1: Log in to Supabase

  1. Go to https://supabase.com
  2. Sign in to your account (or create one if needed)

Step 2: Create or Select Your Project

If you don't have a project:

  1. Click "New Project" button
  2. Enter a project name (e.g., "Productivity Tracker")
  3. Enter a database password (save this!)
  4. Select a region closest to you
  5. Click "Create new project"
  6. Wait 2-3 minutes for setup

If you already have a project:

  • Click on your project from the dashboard

Step 3: Navigate to API Settings

  1. In your project dashboard, look at the left sidebar
  2. Click on "Settings" (gear icon ‚öôÔ∏è at the bottom)
  3. Click on "API" in the settings menu

Step 5: Create Your .env Files

Frontend .env file (in root directory)

Create a file named .env in the root directory:

VITE_SUPABASE_URL=https://your-project-id.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here

Backend .env file (in backend directory)

Create a file named .env in the backend folder:

SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_KEY=your-anon-key-here
SUPABASE_SERVICE_KEY=your-service-role-key-here

‚ö†Ô∏è Important Security Notes

  • Never commit .env files to Git - They're already in .gitignore
  • Never share your service_role key - It has full database access
  • The anon key is safe for frontend use - it's restricted by Row Level Security (RLS)
  • Keep your .env files local - Don't upload them anywhere public
